# Mini-OFBiz实体引擎（SpringBoot4.x+JDK21版）+ RBAC权限系统 需求文档 & 单日任务拆解

# 一、项目整体概述

## 1.1 项目名称

Mini-OFBiz Entity Engine（JDK21 + SpringBoot4.0.6）内置RBAC权限系统

## 1.2 项目定位

基于**JDK21、SpringBoot4.0.6**全新搭建的自研中间件练手项目，复刻 Apache OFBiz 核心实体引擎架构，摒弃 MyBatis、JPA 等第三方持久层框架。以**XML元数据驱动、充血实体模型、统一Delegator代理入口**为核心，实现全自动CRUD、自动DDL建表、动态SQL生成能力。

同时基于自研实体引擎，**从零手写完整企业级RBAC权限系统**（用户、角色、菜单、权限、数据权限），无Security、Shiro框架依赖，纯自研权限校验逻辑。项目兼具底层中间件架构设计与业务权限落地，适配SpringBoot4新版本特性，是差异化极强的后端求职作品集项目。

## 1.3 核心目标

- 技术栈升级：适配 JDK21 新特性、SpringBoot4.0.6 全新自动装配规范，兼容新版本生态

- 复刻OFBiz核心能力：XML实体定义、GenericValue充血实体、Delegator统一数据库操作，实现零DAO、零Mapper、零手写SQL

- 自研完整RBAC权限体系：实现用户-角色-菜单-接口权限关联、权限拦截、数据权限控制

- 实现SpringBoot4.x自定义Starter自动化装配，开箱即用，可独立复用实体引擎与权限模块

- 具备自动DDL建表、动态SQL、事务管理、一级缓存、权限校验全套企业级特性

## 1.4 运行环境（全新升级）

- JDK 21（LTS，启用新版本语法与特性）

- SpringBoot 4.0.6（适配Spring6新规范）

# 二、整体功能需求（全集，新增RBAC模块）

## 2.1 元数据解析能力（核心基础）

- 支持自定义 entitymodel.xml 实体配置文件，兼容OFBiz基础语法，适配新版本解析规范

- 解析实体名、表名、字段、主键、索引、字段语义类型

- 全局内存缓存所有实体元数据，项目启动一次性加载，线程安全适配JDK21并发特性

- 内置OFBiz标准语义类型映射（id/name/date-time/金额/文本等）

## 2.2 充血实体模型能力

- 实现 GenericValue 充血实体类，实现Map结构存储字段值，适配JDK21新集合特性

- 实体自带 create()、store()、remove() 持久化方法，无需外部调用DAO

- 实体绑定Delegator与元数据模型，自动校验字段合法性、非空、长度约束

## 2.3 Delegator统一操作引擎

- 全局唯一数据库操作入口，封装底层JDBC操作，适配SpringBoot4数据源规范

- 基础能力：实体创建、主键查询、新增、更新、删除、条件查询、分页、排序

- 统一异常封装、日志打印，适配新版本Spring异常处理机制

## 2.4 动态SQL引擎

- 根据XML元数据自动生成 INSERT/UPDATE/DELETE/SELECT 标准SQL语句

- 全程使用PreparedStatement预编译，杜绝SQL注入，保证安全性

- 自动适配MySQL字段类型、主键、非空、索引约束

## 2.5 自动DDL能力

- 项目启动自动对比元数据与数据库表结构差异

- 自动创建数据表、新增字段、创建索引，适配RBAC五张核心权限表

- 安全策略：仅新增表、字段、索引，不修改、不删除存量数据，规避数据风险

## 2.6 事务与缓存能力

- 自定义事务管理器，完美整合SpringBoot4.x事务机制，支持@Transactional注解

- 实现线程级一级缓存，同线程重复查询复用内存数据，提升权限查询性能

## 2.7 SpringBoot4.x自动装配

- 适配SpringBoot4全新自动配置机制，自定义Starter场景启动器

- 支持yml配置自定义XML路径、自动DDL开关、权限拦截开关

- 零配置自动注入Delegator、权限校验工具类等核心Bean

## 2.8 自研RBAC权限系统（新增核心业务模块）

完全基于自研Mini-OFBiz实体引擎开发，不依赖任何第三方权限框架，核心包含**5大权限实体、全套权限逻辑、接口拦截、数据权限**

- **五大核心权限实体（XML自定义定义，自动建表）**：
        

    - sys_user：系统用户（账号、密码、状态、创建时间）

    - sys_role：系统角色（角色名、角色标识、排序、状态）

    - sys_user_role：用户角色关联表（一对多关联）

    - sys_menu：菜单/权限表（菜单名、路由、接口权限标识、父级ID、类型）

    - sys_role_menu：角色菜单权限关联表

- **账号认证能力**：自定义密码加密、账号状态校验、登录校验、会话存储

- **权限关联能力**：用户分配多角色、角色分配多菜单权限、权限粒度精确到接口

- **全局接口拦截**：自定义SpringMVC拦截器，未登录拦截、无权限拦截

- **权限注解控制**：自定义@HasPermission、@HasRole注解，实现方法级权限控制

- **基础数据权限**：普通用户仅查询自身数据，管理员查询全部数据

- **权限缓存**：缓存用户角色、权限信息，减少数据库重复查询
